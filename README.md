## How to Submit

### 1. Complete the Challenge

Build a SQL database that passes 100% of SQLLogicTest using one of our seed repos:

- [challenge-seed-rust](https://github.com/vibesql-challenge/challenge-seed-rust) - Rust/Cargo
- [challenge-seed-cpp](https://github.com/vibesql-challenge/challenge-seed-cpp) - C++/CMake
- [challenge-seed-go](https://github.com/vibesql-challenge/challenge-seed-go) - Go

Or start from scratch in any language.

### 2. Open an Issue

[Create a new issue](https://github.com/vibesql-challenge/submissions/issues/new) with:

```
Title: [Submission] <Your Project Name>

Repository: https://github.com/YOUR_USERNAME/YOUR_REPO
Language: <Rust/C++/Go/etc>

Start commit: <hash> (<date>)
100% commit: <hash> (<date>)

Time: <X days>
```

### 3. Verification

We'll verify your submission by checking:

- [ ] Public git history from start to 100%
- [ ] First commit is a fork of a seed repo (or empty init)
- [ ] No code copied from existing databases
- [ ] All 622 SQLLogicTest files pass
- [ ] Timestamps are authentic (not amended)

## Rules

1. **Start fresh** - Fork a seed repo or init empty. Your first commit is day 0.
2. **Build from scratch** - No copying from VibeSQL, SQLite, DuckDB, or other databases
3. **Public repo** - Your git history is your proof
4. **100% pass rate** - All 622 SQLLogicTest files must pass
5. **AI allowed** - Use Claude, Copilot, or any AI tools you want

## FAQ

**Q: Can I use AI assistants?**
A: Yes! This is a *vibe coding* challenge. Use Claude, Copilot, ChatGPT, or any AI tools.

**Q: Can I use libraries?**
A: Yes for utilities (parsing, data structures). No for SQL execution engines.

**Q: What if I find a bug in SQLLogicTest?**
A: Document it in your submission. We'll review case by case.

**Q: Do weekends count?**
A: Yes, calendar days from first commit to 100% commit.
